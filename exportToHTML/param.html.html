    <h1>Parameters and Calculations</h1>
    <p> A parameter is like a variable that can be inputted into functions.</p>
        <div id="simple6"></div>
    <p>Furthermore, a parameter is a variable which the user can adjust, either via a slider or by manipulating something on the graph. The initial value of the parameter is determined by the 'value' of a. The range of the slider is determined by the 'min' and 'max' of a. We can create a sidebar and a slider like this: </p>
        <div id="simple1"></div>
    <p>You can change the precision of a by creating a 'round' command. Here, we round a to the nearest tenth:</p>
            <div id="simple5"></div>
        <p>Drag the slider for a back and forth. Oh no! It displays badly. We can fix this by including a ".toFixed(1)" after it:
            <div id="simple2"></div>
        <p>Parameters may be added together inline:</p>
        <div id="simple3"></div>
        <p>However, it's better to use a calculation to pre-calculate the summed value. A calculation is a function that takes in parameters, based on the format of mathjs (https://mathjs.org/docs/expressions/index.html). We can make calculations like this:</p>
        <div id="simple4"></div>
        <p>Note that calculations work best when all terms are surrounded by parantheses.</p>
        <p>For example, "((params.a)^(params.b))" is more robust than "params.a^params.b". Without parentheses, the expression may be evaluated incorrectly.</p>
